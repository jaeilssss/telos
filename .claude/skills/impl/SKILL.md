현재 프로젝트 루트의 `SPEC.md`를 기준으로 구현을 수행한다.

## 사전 게이트

`SPEC.md`가 현재 작업 디렉토리에 존재하는지 Bash 툴로 확인한다.

존재하지 않으면 **즉시 중단**하고 아래 메시지를 출력한다:

```
🚫 SPEC.md가 없습니다.
구현 전에 /spec으로 인수 기준을 먼저 확정하세요.
```

SPEC.md의 상태가 `draft`이면 마찬가지로 중단한다:

```
🚫 SPEC.md가 아직 draft 상태입니다.
/spec을 완료해 frozen 상태로 만든 뒤 다시 실행하세요.
```

## 모델 선택

SPEC.md가 `frozen` 상태임을 확인했으면 AskUserQuestion 툴로 아래 선택지를 표시한다:

- 질문: "구현에 사용할 모델을 선택하세요."
- 선택지 1 — **Sonnet** (추천): 빠르고 비용 효율적. 대부분의 구현에 적합.
- 선택지 2 — **Opus**: 느리지만 복잡한 추론·설계가 필요한 구현에 적합.

## 구현

선택한 모델로 Agent 툴을 호출해 구현을 위임한다.

- Sonnet 선택 시: `model: "sonnet"`
- Opus 선택 시: `model: "opus"`

Agent 프롬프트에 반드시 포함할 것:
1. 현재 디렉토리의 `SPEC.md` 전체 내용을 Read해서 인수 기준(AC)을 파악할 것
2. 코드베이스를 파악한 뒤 AC를 하나씩 충족하는 방향으로 구현할 것
3. 구현 완료 후 변경한 파일 목록과 각 AC 대응 여부를 요약해서 반환할 것

## 완료 안내

Agent 구현이 끝나면 반드시 아래 메시지를 출력한다:

```
✅ 구현 완료.
/eval 로 인수 기준을 채점하세요.
```
